## 1. 사용자 및 인증 관련

### 1.1 회원가입/로그인
**Q1-1-1) 회원가입 방식은 어떻게 할까요?**B.카카오만
- A) 이메일/비밀번호만
- B) 소셜 로그인 (구글, 카카오 등)만
- C) 이메일/비밀번호 + 소셜 로그인 모두 지원
- D) 학교 이메일로만 가입 가능

📌 **추천**: C (이메일/비밀번호 + 소셜 로그인 모두 지원)
💡 **이유**: 사용자 선택권 제공으로 가입 장벽 낮춤. 학생들은 구글 계정 활용도가 높음

**Q1-1-2) 학생 인증은 어떻게 할까요?** C...추후 A가능성 있음
- A) 학교 이메일 도메인으로 제한 (예: @university.ac.kr)
- B) 학번 입력 및 검증
- C) 별도 인증 절차 없음 (누구나 가입 가능)
- D) 학교 이메일 인증 + 학번 입력

📌 **추천**: A (학교 이메일 도메인으로 제한)
💡 **이유**: 간단하면서도 효과적인 학생 인증. 구현이 쉽고 관리 용이

**Q1-1-3) 사용자 프로필에 어떤 정보를 저장할까요?** 
닉네임+학교명.타겟팅 없이 그냥 자유 참여
- A) 닉네임만
- B) 닉네임 + 학과
- C) 닉네임 + 학과 + 학년
- D) 닉네임 + 학과 + 학년 + 학번

📌 **추천**: C (닉네임 + 학과 + 학년)
💡 **이유**: 설문 타겟팅에 필요한 최소 정보. 학번은 개인정보 우려로 선택사항
